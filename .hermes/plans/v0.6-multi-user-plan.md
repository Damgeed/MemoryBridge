# Memory Bridge v0.6 — Multi-User Architecture

**Goal:** Multiple users/projects on a single Memory Bridge instance, with API key management and data isolation.

## Items

### Phase 1
**Henry 🧠** — Multi-key auth + project isolation
- `api_keys` table (key_hash, label, project_id, is_active, created_at, last_used_at)
- Storage methods: create_key, list_keys, revoke_key, authenticate_key
- `project` field on MemoryEntry and Session models
- Auth middleware checks DB keys + env var fallback
- Project-scoped queries when using a project-bound key

### Phase 2
**Fred 🚀** — Admin API endpoints
- POST /admin/keys (create new key with label + optional project)
- GET /admin/keys (list all keys, hide hashes)
- DELETE /admin/keys/:id (revoke)
- Test all endpoints
