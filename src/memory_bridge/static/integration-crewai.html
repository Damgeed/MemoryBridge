CrewAI Integration

Equip your CrewAI agents with persistent memory tools. Store, recall, and search information across crew runs and sessions.

Overview

The MemoryBridgeTool adapter integrates with CrewAI's tool system, giving your agents the ability to store and retrieve memories from a shared, persistent store. All agents in a crew can access the same memory pool when configured with the same session_id.

✨ Note: You can also use the higher-level MemoryBridgeCrewAI.create_agent() helper for a one-liner agent setup.

Installation

bash
pip install memory-bridge crewai

Quick Start

1

Import and create the memory tool

Create a MemoryBridgeTool instance for your crew.

Python
from memory_bridge.adapters.crewai_adapter import MemoryBridgeTool

memory_tool = MemoryBridgeTool(
    api_key="mb_your-api-key-here",
    base_url="http://localhost:8000",
    session_id="my-crew-session"
)
2

Add the tool to your agents

Pass the memory tool to your CrewAI agents. Each agent can now store and recall information.

Python
from crewai import Agent, Task, Crew

researcher = Agent(
    role="Senior Researcher",
    goal="Uncover cutting-edge findings",
    backstory="You're a meticulous researcher with perfect recall.",
    tools=[memory_tool],
    verbose=True
)

writer = Agent(
    role="Technical Writer",
    goal="Synthesize research into clear documentation",
    backstory="You transform raw data into elegant prose.",
    tools=[memory_tool],  # Shared memory pool!
    verbose=True
)
3

Create tasks and run the crew

Agents automatically use the memory tool during execution.

Python
research_task = Task(
    description="Research the latest advances in AI memory systems",
    expected_output="A bullet-point summary of key findings",
    agent=researcher
)

write_task = Task(
    description="Write a technical blog post based on the research",
    expected_output="A markdown blog post",
    agent=writer
)

crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, write_task],
    verbose=2
)

result = crew.kickoff()

Complete Working Example

A fully runnable CrewAI example with shared memory persistence:

Python
from memory_bridge.adapters.crewai_adapter import MemoryBridgeTool, MemoryBridgeCrewAI
from crewai import Agent, Task, Crew

# Option 1: Add memory tool to existing agents
memory_tool = MemoryBridgeTool(
    api_key="mb_your-api-key",
    base_url="http://localhost:8000",
    session_id="shared-crew-memory"
)

agent = Agent(
    role="Researcher",
    goal="Find information and remember it",
    tools=[memory_tool]
)

# Option 2: Use the one-liner helper
agent2 = MemoryBridgeCrewAI.create_agent(
    role="Researcher",
    goal="Find information",
    api_url="http://localhost:8000",
    api_key="your-key-here"
)

# Store a memory programmatically
memory_tool.store(key="discovery", value={"topic": "memory systems", "importance": "high"})

# Retrieve memories
result = memory_tool.recall(key="discovery")
print("Retrieved:", result)

# Search across all memories
results = memory_tool.search(query="memory systems")

How Memory Flows in a Crew

When multiple agents use the same MemoryBridgeTool instance (or different instances with the same session_id):

  • Shared context — All agents read from the same memory pool, enabling cross-agent knowledge sharing
  • Persistent state — Memories survive crew restarts, server reboots, and multi-day projects
  • Semantic search — Agents can discover relevant memories created by other agents using natural language queries
  • Scoped sessions — Use different session_id values for different projects or clients

⚠️ Production Tip: Use a shared session_id across agents in the same crew, and different session IDs for different crews. This keeps memory contexts isolated while enabling sharing within a crew.
