AutoGen Integration

Wrap AutoGen agents with Memory Bridge for persistent, cross-session memory. Conversation context and learned facts survive beyond individual chats.

Overview

The MemoryBridgeAgent adapter wraps AutoGen's ConversableAgent with persistent memory capabilities. Every remember() and recall() call is backed by the Memory Bridge API, enabling agents to retain knowledge across conversations, sessions, and group chats.

✨ Note: Use the MemoryBridgeAutoGen.create_agent() convenience method for a one-line agent setup with full memory integration.

Installation

bash
pip install memory-bridge pyautogen

Quick Start

1

Import and create a memory-backed agent

Use MemoryBridgeAgent directly or the create_agent helper.

Python
from memory_bridge.adapters.autogen_adapter import MemoryBridgeAgent

agent = MemoryBridgeAgent(
    name="assistant",
    api_key="mb_your-api-key-here",
    base_url="http://localhost:8000",
    session_id="my-autogen-session"
)
2

Store and recall memories

Use the remember and recall methods to persist and retrieve information.

Python
# Store a memory
await agent.remember(
    key="user_preference",
    value={"language": "Python", "framework": "FastAPI"}
)

# Recall a memory
prefs = await agent.recall(key="user_preference")
print(prefs)  # {'language': 'Python', 'framework': 'FastAPI'}

# Semantic search
results = await agent.search(query="user framework preferences")
3

Use in an AutoGen conversation

Agents with memory naturally persist context across messages.

Python
from autogen import ConversableAgent, UserProxyAgent

user_proxy = UserProxyAgent(
    name="User",
    human_input_mode="NEVER",
    code_execution_config=False
)

# Start a conversation — agent can recall past interactions
await user_proxy.a_initiate_chat(
    agent,
    message="Do you remember my preferred framework?"
)

Complete Working Example

A complete AutoGen agent with Memory Bridge persistence:

Python
import asyncio
from memory_bridge.adapters.autogen_adapter import MemoryBridgeAgent, MemoryBridgeAutoGen
from autogen import AssistantAgent, UserProxyAgent, ConversableAgent

async def main():
    # Option 1: Direct MemoryBridgeAgent
    memory_agent = MemoryBridgeAgent(
        name="researcher",
        api_key="mb_your-api-key",
        base_url="http://localhost:8000",
        session_id="autogen-demo"
    )

    # Store some knowledge
    await memory_agent.remember(
        key="project_context",
        value={"project": "Memory Bridge", "status": "active"}
    )

    # Option 2: One-liner with create_agent
    agent2 = MemoryBridgeAutoGen.create_agent(
        name="researcher",
        llm_config={"config_list": [{"model": "gpt-4", "api_key": "sk-..."}]},
        api_url="http://localhost:8000",
        api_key="your-key-here"
    )

    # Recall stored memory across sessions
    context = await memory_agent.recall(key="project_context")
    print("Recalled:", context)

if __name__ == "__main__":
    asyncio.run(main())

Multi-Agent Memory Sharing

Memory Bridge enables cross-agent memory in AutoGen group chats:

  • Shared session — Agents with the same session_id share a memory pool
  • Agent-specific keys — Use prefixed keys (e.g., "researcher:findings") for per-agent namespacing
  • Cross-session continuity — Memories persist even when all agents restart, enabling long-running research projects
  • Semantic discovery — Agents can search for relevant memories created by other agents using natural language

⚠️ Production Tip: For multi-agent scenarios, set the same project name across agents and use different session_id values for different conversations. This enables cross-agent retrieval within a project while isolating conversations.
