LangChain & LangGraph Integration

Use Memory Bridge as a drop-in checkpoint saver for LangGraph state graphs. Agents automatically persist state across sessions and conversations.

Overview

The MemoryBridgeSaver adapter implements LangGraph's BaseCheckpointSaver interface, allowing you to pass it directly to graph.compile(checkpointer=...). All state checkpoints are persisted to Memory Bridge and retrievable across sessions by session_id.

✨ Note: You can also use the higher-level MemoryBridgeLangGraph wrapper for a simpler API that manages the graph lifecycle for you.

Installation

Install the Memory Bridge SDK (or copy the adapter directly):

bash
pip install memory-bridge

Quick Start

1

Import the adapter

Import the LangGraph-compatible checkpoint saver.

Python
from memory_bridge.adapters.langgraph_adapter import MemoryBridgeSaver
from langgraph.graph import StateGraph
2

Configure the checkpointer

Create a saver instance with your Memory Bridge API credentials.

Python
checkpointer = MemoryBridgeSaver(
    api_key="mb_your-api-key-here",
    base_url="http://localhost:8000",
    project="my-agent-project",
    session_id="my-session"
)
3

Compile your graph with the checkpointer

Pass the saver to graph.compile() — all state transitions are now persisted.

Python
class AgentState(TypedDict):
    messages: list
    context: str

def build_graph():
    graph = StateGraph(AgentState)
    # ... add nodes and edges ...
    app = graph.compile(checkpointer=checkpointer)
    return app
4

Store and retrieve memories

Use the memory API directly to store and recall facts across sessions.

Python
# Store a memory
await checkpointer.store_memory(
    key="user_preference",
    value={"theme": "dark", "language": "en"}
)

# Retrieve memories
result = await checkpointer.recall(
    key="user_preference"
)

# Search semantically
results = await checkpointer.search(
    query="user theme settings",
    limit=5
)

Complete Working Example

A fully runnable LangGraph agent with Memory Bridge persistence:

Python
import asyncio
from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from memory_bridge.adapters.langgraph_adapter import MemoryBridgeSaver

class AgentState(TypedDict):
    messages: list
    context: str

async def main():
    # 1. Create the checkpointer
    checkpointer = MemoryBridgeSaver(
        api_key="mb_your-api-key",
        base_url="http://localhost:8000",
        session_id="demo-session"
    )

    # 2. Build a simple LangGraph
    builder = StateGraph(AgentState)
    builder.add_node("agent", lambda state: {
        "messages": state["messages"] + ["Hello from Memory Bridge!"],
        "context": state.get("context", "")
    })
    builder.add_edge(START, "agent")
    builder.add_edge("agent", END)
    app = builder.compile(checkpointer=checkpointer)

    # 3. Store user info
    await checkpointer.store_memory(
        key="user_info",
        value={"name": "Alice", "tier": "pro"}
    )

    # 4. Run the graph
    result = await app.ainvoke(
        {"messages": [], "context": ""},
        config={"configurable": {"thread_id": "thread-1"}}
    )
    print(result)

    # 5. Retrieve stored memory
    memory = await checkpointer.recall(key="user_info")
    print("Stored memory:", memory)

if __name__ == "__main__":
    asyncio.run(main())

Using the Higher-Level Wrapper

For convenience, you can also use MemoryBridgeLangGraph which wraps both the graph and checkpointer:

Python
from memory_bridge.adapters.langgraph_adapter import MemoryBridgeLangGraph
from langgraph.graph import StateGraph

graph = MemoryBridgeLangGraph(
    api_url="http://localhost:8000",
    api_key="your-key-here",
    session_id="my-session"
)

# Use as a LangGraph StateGraph
app = graph.compile()

API Reference

The MemoryBridgeSaver adapter supports the following operations:

  • put(config, checkpoint, metadata) — Store a checkpoint (LangGraph protocol)
  • get(config) — Retrieve the latest checkpoint for a thread
  • store_memory(key, value) — Store an arbitrary key-value memory
  • recall(key) — Retrieve a memory by exact key match
  • search(query, limit, filters) — Semantic search across all memories

⚠️ Production Tip: Use a unique session_id per user or conversation to isolate memory scopes. In production, set base_url to your deployed Memory Bridge instance.
