It's one shared memory for all your agents

You're building with AI agents. A researcher, a coder, a reviewer. Each one is sharp — but they don't share a brain. Every new session is a fresh start. You're the only one holding context in your head.

1

Agent stores a memory

Henry explores, discovers a fact, writes it to Memory Bridge. Session doesn't matter — the memory persists.

2

Another agent picks it up

Fred starts a new session, queries Memory Bridge. The fact Henry stored is there — no hand-holding needed.

3

Handoff with full context

Alice hands off to Bob with a structured handoff packet. Bob gets everything — context, unresolved issues, history.

⚡ See It In Action

💾 Henry stores a memory
# Henry discovers the user's preferences
$ curl -X POST /memories \
  -H "Authorization: Bearer <key>" \
  -d '{ "session_id": "session-1", "agent_id": "henry", "key": "user_preferences", "value": "{\"python\":\"3.12\",\"theme\":\"dark\"}", "tags": ["user-profile", "important"] }'

# Response: ✓ memory stored with ID mem_abc123
✅ Memory persisted. Henry's session can end — the knowledge stays.
🔍 Fred queries in a brand new session
# Fred starts fresh — asks "what do we know?"
$ curl /memories/query \
  -H "Authorization: Bearer <key>" \
  -d '{"tags": ["user-profile", "important"]}'

{ "entries": [{ "agent_id": "henry", "key": "user_preferences", "value": {"python": "3.12", "theme": "dark"}, "tags": ["user-profile", "important"] }] }
✅ Fred has Henry's knowledge without a single conversation between them.
🤝 Alice hands off to Bob
# Alice passes context to Bob in one call
$ curl -X POST /handoff/execute \
  -H "Authorization: Bearer <key>" \
  -d '{ "from_agent_id": "alice", "to_agent_id": "bob", "session_id": "session-1", "context": {"task": "implement login", "findings": "use OAuth2"} }'
✅ Bob gets context + guardrails. No lost context, no "what were we doing?"
