Memory Bridge

Cross-session memory persistence for multi-agent AI teams

https://memory-bridge-app-production.up.railway.app
🧠

Session Persistence

Store and retrieve agent context across sessions. Memories persist beyond individual conversations.

🏷️

Memory Tagging

Organize memories with tags for selective retrieval. Filter by agent, session, or content type.

🔄

Agent Handoff

Pass context between agents with structured guardrails. From agent A to agent B, seamlessly.

🔌

Framework Adapters

Plug into LangGraph, AutoGen, CrewAI, or use the raw REST API. Python SDK included.

🔒

Secure by Default

API key + JWT auth. Multi-tenant isolation with schema-per-tenant and RLS on PostgreSQL.

📊

Built-in Analytics

Prometheus metrics, audit logging, webhook subscriptions, and tier-based rate limiting.

⚡ Quick Start

# Install the Python SDK
pip install memory-bridge

# Or use the REST API directly
curl -H "Authorization: Bearer <your-api-key>" \
  https://memory-bridge-app-production.up.railway.app/health

# Store agent memory
curl -X POST https://memory-bridge-app-production.up.railway.app/memories \
  -H "Authorization: Bearer <your-api-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "s1",
    "agent_id": "alice",
    "key": "user_name",
    "value": "Alice"
  }'

📋 API Endpoints

MethodEndpointDescription
GET/healthService health & operational metrics
POST/memoriesCreate a memory entry
GET/memories/{id}Get a memory by ID
POST/memories/queryQuery memories with filters
GET/memories/searchFull-text search across memories
POST/sessionsCreate a session
POST/handoff/executeExecute agent-to-agent handoff
POST/webhooksSubscribe to memory events
